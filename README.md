# Мультибиржевый P2P-бот (Bybit, Bitget, HTX, MEXC)

Бот для клиентов: человек пишет боту, жмёт "Купить" или "Продать", выбирает биржу
и получает ссылку на твоё объявление. Тебе приходит уведомление о переходе клиента,
а для Bybit и Bitget - ещё и автоматическое уведомление о реальном новом ордере
(с ником и суммой), потому что у этих двух бирж есть открытый P2P API.

## Важно понимать сразу: разные биржи = разные возможности

| Биржа | Что делает бот |
|---|---|
| **Bybit** | Сам находит твоё активное объявление и даёт на него ссылку. Сам уведомляет о новом ордере (ник, сумма) - через официальный P2P Open API. |
| **Bitget** | То же самое, но через P2P API для верифицированных мерчантов. **Точный путь эндпоинта для списка ордеров может отличаться** - см. раздел "Если Bitget не работает" ниже. |
| **HTX** | Открытого API для управления P2P нет. Ты сам вписываешь боту ссылку на своё объявление и цену командами (`/setlink`, `/setprice`), а когда клиент нажмёт кнопку - бот пришлёт тебе уведомление о переходе (но не о реальном ордере - это нужно проверять в приложении HTX). |
| **MEXC** | Аналогично HTX - ручной режим. |

## Установка

```
pip install -r requirements.txt
cp .env.example .env
```

Заполни `.env`:
- `TELEGRAM_BOT_TOKEN` - токен от @BotFather
- `ADMIN_CHAT_ID` - твой личный chat_id (узнать у @userinfobot)
- Для Bybit: `BYBIT_API_KEY`, `BYBIT_API_SECRET` (права: P2P, без Withdraw)
- Для Bitget: `BITGET_API_KEY`, `BITGET_API_SECRET`, `BITGET_API_PASSPHRASE` -
  создаются в API Management, нужен статус верифицированного P2P-мерчанта
- Для HTX/MEXC ключи не нужны - просто оставь `HTX_ENABLED=True` / `MEXC_ENABLED=True`

Если какую-то биржу пока не используешь - поставь у неё `_ENABLED=False` в `.env`.

## Запуск

```
python bot.py
```

## Первая настройка ручных бирж (HTX, MEXC)

После запуска бота напиши ему сам (с аккаунта, чей chat_id указан в `ADMIN_CHAT_ID`):

```
/setlink htx buy https://www.htx.com/ссылка_на_твоё_объявление
/setprice htx buy 95.50
/setlink htx sell https://www.htx.com/ссылка_на_другое_объявление
/setprice htx sell 94.80
```

То же самое для `mexc`. Проверить, что всё сохранилось - командой `/status`.

## Команды

**Для клиентов** (любой человек):
- `/start` - показывает кнопки "Купить" / "Продать", дальше бот сам ведёт по шагам

**Для тебя (администратора)**:
- `/status` - показать все биржи и их текущие объявления/цены
- `/setlink <биржа> <buy|sell> <ссылка>` - задать ссылку (только для htx, mexc)
- `/setprice <биржа> <buy|sell> <цена>` - задать цену (только для htx, mexc)

## Если Bitget не работает (ошибка 404/401 при проверке ордеров)

Путь эндпоинта для получения списка P2P-ордеров у Bitget может отличаться в зависимости
от версии API. В файле `exchanges/bitget_ex.py` есть строка:
```python
ORDERS_PATH = "/api/v2/p2p/orderList"
```
Зайди в свой Bitget API Dashboard -> раздел P2P API документация (доступен только
верифицированным мерчантам) и найди точный путь для "Get Merchant P2P Orders" -
пришли мне его, я поправлю код. То же самое, если понадобится реализовать
автоматическое изменение цены объявлений на Bitget (сейчас бот только читает
объявления и ордера, не меняет цену).

## Про ссылку на объявление Bybit

В `exchanges/bybit_ex.py` ссылка собирается по шаблону:
```
https://www.bybit.com/fiat/trade/otc/?actionType=...&token=...&fiat=...&itemId=...
```
Это стандартный формат ссылки на P2P-раздел Bybit с параметрами объявления - но
обязательно проверь на реальном объявлении, что ссылка открывает именно нужную
карточку, а не просто общий список объявлений. Если ссылка ведёт не туда - пришли
мне, как выглядит настоящая ссылка на конкретное объявление (скопируй её из приложения
Bybit через "Поделиться" на своём объявлении), и я поправлю шаблон.

## Загрузка на Railway (чтобы работал 24/7)

Точно так же, как описано в предыдущей инструкции для P2P-бота:
1. `git init && git add . && git commit -m "multi exchange bot"`, создать репозиторий
   на GitHub и запушить код (`.env` и `data.json` не попадут туда благодаря `.gitignore`)
2. На railway.com: New Project -> Deploy from GitHub repo -> выбрать репозиторий
3. Во вкладке Variables вручную вписать все переменные из `.env`
4. `Procfile` уже добавлен (`worker: python bot.py`) - Railway должен подхватить его сам,
   иначе вручную задать Start Command в Settings -> Deploy
5. Проверить логи - должна появиться строка "Мультибиржевый P2P-бот запущен!"

**Важный нюанс с `data.json`:** этот файл хранит ссылки/цены для ручных бирж и список
уже увиденных ордеров. На Railway при каждом новом деплое файловая система может
сбрасываться, и настройки `/setlink`/`/setprice` придётся вбивать заново. Если это
станет проблемой - можно перенести хранение в Railway PostgreSQL (бесплатный плагин),
скажи, если понадобится - подключу.
